# 5.1 Соображения по обеспечению Web‑безопасности (Web Security Considerations)

## Общий обзор

World Wide Web — это клиент/серверное приложение, работающее поверх Интернета и TCP/IP‑интранетов. Хотя классические методы сетевой безопасности применимы и здесь, Web добавляет **свои собственные угрозы и сложности**.

## Основные проблемы безопасности Web

* **Интернет — двусторонний канал.** В отличие от старых односторонних систем (телетекст, автоответчики, факс‑back), Web‑серверы могут быть атакованы извне.
* **Высокая видимость.** Web‑страницы — это корпоративная витрина. Взлом может привести к потере репутации и денежным потерям.
* **Сложность ПО.** Web‑серверы, браузеры и Web‑приложения просты для пользователя, но чрезвычайно сложны внутри → это скрывает множество уязвимостей.
* **Точка опоры для атак.** Скомпрометированный Web‑сервер может дать злоумышленнику доступ к внутренней корпоративной сети.
* **Необученные пользователи.** Обычные клиенты Web‑сервисов не знают о рисках и не могут защищаться.

---

# Типы угроз Web (Web Security Threats)

Угрозы можно разделить по двум группам:

## **1. Пассивные атаки (Passive Attacks)**

* Прослушивание трафика между браузером и сервером.
* Получение доступа к информации, которая должна быть закрыта.

## **2. Активные атаки (Active Attacks)**

* Имперсонация пользователя (impersonation).
* Изменение сообщений между клиентом и сервером.
* Подмена или удаление контента на Web‑сервере.

## **Классификация по месту угрозы**

| Локация угрозы  | Примеры                                             |
| --------------- | --------------------------------------------------- |
| **Web сервер**  | Уязвимости в ПО сервера, взлом CMS, подмена страниц |
| **Web браузер** | XSS, вредоносные скрипты, уязвимости расширений     |
| **Трафик**      | MITM, перехват, модификация пакетов                 |

# Подходы к защите Web‑трафика (Web Traffic Security Approaches)

Существует несколько способов обеспечить безопасность Web‑трафика. Они отличаются **уровнем TCP/IP‑стека**, на котором работают.

---

# 1. Уровень сети — **IPsec** (Network Level Security)

Работает на уровне **IP**.

### Преимущества:

* Прозрачно для пользователей и приложений.
* Универсальный механизм защиты.
* Можно фильтровать трафик (не весь трафик должен идти через IPsec).
  
---

# 2. Уровень транспорта — **SSL/TLS** (Transport Level Security)

Работает **над TCP**, но под HTTP/SMTP/FTP.

### Примеры:

* **SSL (Secure Sockets Layer)**
* **TLS (Transport Layer Security)** — современный стандарт

### Два варианта реализации:

1. Как часть системного стека (прозрачно).
2. Как часть конкретных приложений (например, браузеры уже включают TLS).


---

# 3. Уровень приложений — Application‑Level Security

Безопасность встроена в само приложение.

### Примеры:

* **S/MIME** — шифрование электронных писем
* **Kerberos** — аутентификация
* Безопасность, встроенная в конкретные Web‑приложения


### Преимущество:

* Решение можно адаптировать под конкретные требования приложения.

---

# Сводная таблица подходов

| Уровень               | Технологии                     | Описание                                                  |
| --------------------- | ------------------------------ | --------------------------------------------------------- |
| **Network Level**     | IPsec                          | Защита всего IP‑трафика; прозрачно; общий подход          |
| **Transport Level**   | SSL/TLS                        | Защита TCP‑протоколов (HTTPS, SMTPS); широко используется |
| **Application Level** | S/MIME, Kerberos, app‑security | Гибкая защита под конкретное приложение                   |

---

# Итог

Web‑безопасность можно реализовать на разных уровнях. Корпорации часто используют несколько уровней одновременно (например, **IPsec + TLS**). Для обычных пользователей основой является **TLS (HTTPS)**, а корпоративные сети могут использовать IPsec для защиты всей инфраструктуры.
